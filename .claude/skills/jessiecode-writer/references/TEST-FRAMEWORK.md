# 测试用例框架

> 用于验证 JessieCode Writer 技能生成代码的质量

---

## 一、语法正确性测试

### 1.1 对象字面量语法

- [ ] 所有对象使用 `<< >>` 而非 `{}`
- [ ] 属性之间使用逗号分隔
- [ ] 属性值使用正确的引号（单引号）

**测试用例**：
```jessiecode
// 正确
A = point(1, 2) << strokeColor: 'red', size: 5 >>;

// 错误（应避免）
A = point(1, 2) { strokeColor: 'red', size: 5 };
```

### 1.2 函数调用语法

- [ ] 所有函数调用使用括号
- [ ] 括号内不加多余空格

**测试用例**：
```jessiecode
// 正确
A = point(1, 2);
B = point(3, 4);

// 错误（应避免）
A = point 1, 2;
B = point( 1, 2 );
```

### 1.3 属性访问语法

- [ ] 使用点号 `.` 访问属性
- [ ] 不使用方括号 `[]` 访问属性

**测试用例**：
```jessiecode
// 正确
A.strokeColor = 'red';
A.size = 5;

// 错误（应避免）
A['strokeColor'] = 'red';
```

### 1.4 幂运算符

- [ ] 使用 `^` 而非 `**` 进行幂运算

**测试用例**：
```jessiecode
// 正确
distance = sqrt((x1-x2)^2 + (y1-y2)^2);

// 错误（应避免，虽然部分版本支持）
distance = sqrt((x1-x2)**2 + (y1-y2)**2);
```

## 二、几何构造覆盖测试

### 2.1 三角形构造

- [ ] 外心（Circumcenter）- 使用 `perpendicular` + `intersection`
- [ ] 内心（Incenter）- 使用 `bisector` + `intersection`
- [ ] 重心（Centroid）- 使用 `midpoint` + `intersection`
- [ ] 垂心（Orthocenter）- 使用 `perpendicularsegment` + `intersection`

**测试输入**：
- "生成三角形外心的代码"
- "如何构造三角形的内心？"
- "演示三角形重心的构造"
- "画出三角形的垂心"

### 2.2 圆构造

- [ ] 基本圆 - 使用 `circle(center, point)`
- [ ] 同心圆 - 多个圆共享同一圆心
- [ ] 相切圆 - 使用 `tangent` 或几何构造
- [ ] 外接圆 - 使用 `circumcircle(A, B, C)`
- [ ] 内切圆 - 使用 `incircle(A, B, C)`

**测试输入**：
- "创建一个圆心在原点，经过点 (2,0) 的圆"
- "生成两个同心圆"
- "画一个三角形的外接圆"

### 2.3 函数图构造

- [ ] 一次函数 - `functiongraph(f, x1, x2)`
- [ ] 二次函数 - 抛物线
- [ ] 三角函数 - sin, cos, tan
- [ ] 带滑块的动态函数

**测试输入**：
- "绘制函数 y = x^2 的图像"
- "生成一个带滑块的二次函数图像"
- "画出 sin(x) 在 [-π, π] 的图像"

## 四、代码风格测试

### 4.1 代码格式化

- [ ] 赋值运算符两侧加空格：`A = point(1, 2);`
- [ ] 函数调用括号内不加空格：`point(1, 2)`
- [ ] 属性块格式统一
- [ ] 注释前保留两个空格

**测试用例**：
```jessiecode
// 标准格式
A = point(1, 2) <<
    strokeColor: 'red',
    fillColor: 'pink',
    size: 5
>>;

A = point(1, 2);  // 创建点 A
```

### 4.2 注释规范

- [ ] 关键步骤有注释说明
- [ ] 注释使用 `//` 格式
- [ ] 注释语言与用户请求语言一致
